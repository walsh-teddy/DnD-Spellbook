API

For this, I am using the D&D 5e API, and here is the documentation.

Major Features

  1. Filter by class, spell level, and school of magic
  2. Save to a spellbook
  3. Input a character's level, class, ability scores and subclass to learn how many spells they can have and how many spellslots they have
  4. Create, submit, and browse custom homebrew spells (I'll probably add my own...)

Noteworthy

This app utilized many different aspects of Javascript, such as fetch(), the Bulma CSS framework, custom web components, faunt awesome icons, local storage, and cloud storage using Firebase.

Grading

For this app, I believe I met every grading requierment with the exception of stylizing it slightly more. Outside of getting more unique and fitting CSS, my app is completely functional. There are no errors that run, my code is neat, commented, and descriptive. I have 4 web components (header, navbar, spellcard, and footer), I switched it from using XHR to fetch(), my spellcards are readable and neat, favoriting works both in saving locally and adding it to the cloud, and UI states are recorded whenever they are changed. I also created a community page to display every spell that's been chosen on anyone's spellbook using firebase.


In terms of going above and beyond, I added a full extra feature to calculate the spell list restrictions for your character, creating custom functions for each spellcasting class in D&D. I also made a long function to properly format the spell information to match exactly (outside of font and color) the way it would apear in a D&D book. Lastly, I made a system to automatically create a node in the firebase data when there was a spellcard shown that didn't exist yet so that if the API is ever updated or the data gets cleared, the app will still run fine without any errors. I also display the spell's pick count recorded on firebase even if its not on the community page. The API I'm using doesn't have a way to filter a class' spell list by spell level and school, so instead I had to make a system that got 2 differetn JSON objects, 1 of the class' entire spell list, and one of every spell in every spell list that matches the filter, and then make a new JSON object of just the overlapping spells.
